Copiloting the Copilots: Fusing Large Language Models with Completion Engines for Automated Program Repair

✍️ Yuxiang Wei, Chunqiu Steven Xia, Lingming Zhang 📅 2023-09-01 원문 보기 →

📝 요약

**한 줄 요약**: LLM의 패치 생성 과정에서 프로그래밍 언어의 문법적 제약을 활용하는 Completion Engine을 도입하여 더 유효한 패치를 생성하는 Repilot 프레임워크를 제안

**문제 정의**:
- 기존 LLM 기반 자동 프로그램 수정 기법들이 프로그램을 단순한 토큰 시퀀스로만 처리하여 프로그래밍 언어의 의미적 제약을 무시함
- 이로 인해 문법적으로 부정확한 패치가 대량 생성되어 실용성이 떨어지는 문제

**핵심 기여**:
1. **Completion Engine과 LLM의 협력적 패치 생성**: LLM의 autoregressive 생성 과정에서 Completion Engine이 실시간으로 개입하여 문법적 제약을 적용
2. **이중 필터링 메커니즘**: (1) LLM이 제안한 부적절한 토큰 제거, (2) Completion Engine의 제안에 기반한 능동적 토큰 완성
3. **Defects4j 1.2/2.0에서 기존 SOTA 대비 27%/47% 성능 향상** 달성

**의의**:
- **Neurosymbolic 접근법의 구체적 구현**: 심볼릭 제약(문법 규칙)과 신경망 모델(LLM)의 실질적 융합 사례 제시
- **제약 기반 프로그램 분석의 새로운 활용법**: 기존의 정적 분석 제약을 LLM 생성 과정에 직접 통합하는 방법론 제공
- **APR 연구의 실용성 개선**: 문법적 유효성 보장을 통해 생성된 패치의 품질 향상, 실제 개발 환경에서의 적용 가능성 증대

📄 원문 초록

During Automated Program Repair (APR), it can be challenging to synthesize correct patches for real-world systems in general-purpose programming languages. Recent Large Language Models (LLMs) have been shown to be helpful "copilots" in assisting developers with various coding tasks, and have also been directly applied for patch synthesis. However, most LLMs treat programs as sequences of tokens, meaning that they are ignorant of the underlying semantics constraints of the target programming language. This results in plenty of statically invalid generated patches, impeding the practicality of the technique. Therefore, we propose Repilot, a general code generation framework to further copilot the AI "copilots" (i.e., LLMs) by synthesizing more valid patches during the repair process. Our key insight is that many LLMs produce outputs autoregressively (i.e., token by token), resembling human writing programs, which can be significantly boosted and guided through a Completion Engine. Repilot synergistically synthesizes a candidate patch through the interaction between an LLM and a Completion Engine, which 1) prunes away infeasible tokens suggested by the LLM and 2) proactively completes the token based on the suggestions provided by the Completion Engine. Our evaluation on a subset of the widely-used Defects4j 1.2 and 2.0 datasets shows that Repilot outperforms state-of-the-art techniques by fixing 27% and 47% more bugs, respectively.
