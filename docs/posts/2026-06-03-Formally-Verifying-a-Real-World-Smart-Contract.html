Formally Verifying a Real World Smart Contract

✍️ Alexandre Mota, Fei Yang, Cristiano Teixeira 📅 2023-07-05 원문 보기 →

📝 요약

**한 줄 요약**: 실제 배포된 최신 Solidity 스마트 컨트랙트에 대해 형식 검증이 가능한 도구를 찾고 평가한 실용적 연구입니다.

**문제 정의**:
스마트 컨트랙트는 배포 후 수정이 어렵고 오류 시 막대한 금전적 손실을 초래할 수 있어 일반적인 테스팅보다 높은 수준의 정확성 보장이 필요하지만, 실제 운영 환경의 최신 Solidity 버전으로 작성된 스마트 컨트랙트를 형식적으로 검증할 수 있는 실용적 도구와 방법론이 부족한 상황을 해결하고자 했습니다.

**핵심 기여**:
- 실제 배포된 스마트 컨트랙트를 대상으로 한 형식 검증 도구들의 실용성 평가
- 최신 Solidity 버전과 실제 산업 환경에서의 형식 검증 적용 가능성 검토
- 이론적 형식 검증과 실제 구현 간의 격차 분석 및 실무 적용 관점의 도구 선정 기준 제시

**의의**:
스마트 컨트랙트 보안 연구에서 형식 검증의 실용적 적용 방안을 제시한 중요한 연구입니다. 특히 바이트코드 분석이나 시그니처 분석과 함께 형식 검증을 활용한 종합적 보안 검증 체계 구축에 직접적인 참고자료가 될 수 있으며, 연구자의 스마트 컨트랙트 보안 검증 도구 개발이나 실제 컨트랙트 분석 시 검증 도구 선택과 적용 전략 수립에 실무적 가이드라인을 제공합니다.

📄 원문 초록

Nowadays, smart contracts have become increasingly popular and, as with software development in general, testing is the standard method for verifying their correctness. However, smart contracts require a higher level of certainty regarding correctness because they are diffcult to modify once deployed and errors can result in significant financial losses. Therefore, formal verification is essential. In this article, we present our search for a tool capable of formally verifying a real-world smart contract written in a recent version of Solidity.
