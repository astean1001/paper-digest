Prompt Engineering vs. Fine-Tuning for LLM-Based Vulnerability Detection in Solana and Algorand Smart Contracts

✍️ Biagio Boi, Christian Esposito 📅 2025-11-14 원문 보기 →

📝 요약

**한 줄 요약**: Ethereum 외 블록체인 플랫폼(Solana, Algorand)의 스마트 컨트랙트 취약점 탐지에서 대규모 언어모델(LLM) 기반 프롬프트 엔지니어링과 파인튜닝 방법을 비교 분석한 연구입니다.

**문제 정의**:
기존 스마트 컨트랙트 취약점 탐지 연구가 주로 Ethereum(EVM) 생태계에 집중되어 있어, Solana(Rust)와 Algorand(PyTeal) 같은 non-EVM 플랫폼의 취약점 탐지 방법이 부족하며, 특히 이러한 플랫폼용 라벨링된 데이터셋이 존재하지 않는 문제를 해결하고자 했습니다.

**핵심 기여**:
1. **합성 데이터셋 구축**: OWASP 기반 취약점 분류체계를 활용해 Rust(Solana)와 PyTeal(Algorand) 스마트 컨트랙트 코드 스니펫의 주석 처리된 합성 데이터셋을 설계
2. **3가지 LLM 구성 비교**: 프롬프트 엔지니어링, 파인튜닝, 하이브리드 방식의 성능을 다양한 취약점 카테고리별로 평가
3. **플랫폼별 특성 분석**: Solana와 Algorand의 아키텍처 차이가 취약점 발현과 탐지에 미치는 영향을 분석하고 플랫폼별 매핑 제공

**의의**:
LLM-프로그램 분석 융합 연구에 중요한 벤치마크를 제공하며, 특히 프롬프트 엔지니어링이 일반적 견고성을, 파인튜닝이 TEAL 같은 의미적으로 제한된 언어에서 정밀도/재현율 향상을 보인다는 발견은 도메인별 LLM 적용 전략 수립에 실용적 가이드라인을 제시합니다. 다양한 블록체인 플랫폼으로의 확장 가능성도 보여줍니다.

📄 원문 초록

Smart contracts have emerged as key components within decentralized environments, enabling the automation of transactions through self-executing programs. While these innovations offer significant advantages, they also present potential drawbacks if the smart contract code is not carefully designed and implemented. This paper investigates the capability of large language models (LLMs) to detect OWASP-inspired vulnerabilities in smart contracts beyond the Ethereum Virtual Machine (EVM) ecosystem, focusing specifically on Solana and Algorand. Given the lack of labeled datasets for non-EVM platforms, we design a synthetic dataset of annotated smart contract snippets in Rust (for Solana) and PyTeal (for Algorand), structured around a vulnerability taxonomy derived from OWASP. We evaluate LLMs under three configurations: prompt engineering, fine-tuning, and a hybrid of both, comparing their performance on different vulnerability categories. Experimental results show that prompt engineering achieves general robustness, while fine-tuning improves precision and recall on less semantically rich languages such as TEAL. Additionally, we analyze how the architectural differences of Solana and Algorand influence the manifestation and detectability of vulnerabilities, offering platform-specific mappings that highlight limitations in existing security tooling.
